NexusIQ REST API v3.2 — Technical Reference

This document provides a complete reference for the NexusIQ REST API v3.2, including authentication, rate limiting, endpoint descriptions, request/response schemas, error codes, and code examples. Base URL: https://api.nexusiq.io/v3. All endpoints require HTTPS.

Authentication

NexusIQ supports two authentication mechanisms: Bearer tokens (recommended for server-side integrations) and API keys (for lightweight scripts and webhooks). All API requests must include an Authorization header.

Authorization: Bearer eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...

Tokens are issued via the OAuth 2.0 client credentials flow. Token lifetime is 3600 seconds. API keys are issued from the Settings > API Keys console and never expire unless revoked.

Rate Limiting

Rate limits are enforced per API key at the account level. The default tier allows 1,000 requests per minute. Enterprise accounts may request elevated limits. Rate limit headers are included in every response.

Endpoints

Documents

POST /documents

Ingest a new document into the pipeline. Accepts multipart/form-data with the file and metadata. Returns a job_id for tracking pipeline progress via WebSocket or polling.

curl -X POST https://api.nexusiq.io/v3/documents   -H "Authorization: Bearer TOKEN"   -F "file=@report.pdf"   -F "pipeline=custom"   -F "metadata={"source":"upload","tags":["medical","trial"]}"

Response (202 Accepted):

{
  "job_id": "j_01HXYZ1234",
  "status": "queued",
  "pipeline": "custom",
  "created_at": "2026-05-17T10:00:00Z",
  "estimated_duration_s": 45
}

GET /documents/{doc_id}

Retrieve metadata and extraction results for a previously ingested document.

curl -X GET https://api.nexusiq.io/v3/documents/doc_abc123   -H "Authorization: Bearer TOKEN"

GET /documents

List all documents with optional filters. Supports pagination via cursor-based navigation.

ParameterTypeRequiredDescription
limitintegerNoResults per page (default 20, max 100)
cursorstringNoPagination cursor from previous response
pipelinestringNoFilter by pipeline: custom | docling
statusstringNoFilter by status: queued | running | completed | error
fromISO8601NoFilter by created_at >= from
toISO8601NoFilter by created_at <= to

DELETE /documents/{doc_id}

Delete a document and all associated chunks, vectors, and metadata from the store. This operation is irreversible. Returns 204 No Content on success.

Search / Retrieval

POST /search

Perform hybrid (dense + sparse) retrieval over the indexed document corpus. Supports dense vector search (ANN), BM25 sparse retrieval, and RRF fusion re-ranking.

curl -X POST https://api.nexusiq.io/v3/search   -H "Authorization: Bearer TOKEN"   -H "Content-Type: application/json"   -d '{
    "query": "What was the primary endpoint result for XR-2041?",
    "top_k": 10,
    "pipeline": "custom",
    "filters": {"doc_type": "research_paper", "domain": "medical"},
    "rerank": true,
    "include_metadata": true
  }'

Jobs

GET /jobs/{job_id}

Poll job status. For real-time updates, use the WebSocket endpoint instead.

WebSocket ws://api.nexusiq.io/v3/ws/{job_id}

Connect to receive real-time stage events as the pipeline processes a document. Each message is a JSON object conforming to the StageEvent schema.

SDK Examples

Python

from nexusiq import NexusIQ
client = NexusIQ(api_key="nxiq_live_xxxxx")
job = client.documents.ingest("report.pdf", pipeline="custom")
result = job.wait()  # blocks until pipeline completes
chunks = result.chunks
print(f"Extracted {len(chunks)} chunks from {result.page_count} pages")

Node.js

import { NexusIQ } from '@nexusiq/sdk';
const client = new NexusIQ({ apiKey: process.env.NEXUSIQ_KEY });
const job = await client.documents.ingest(fs.createReadStream('report.pdf'));
const result = await job.wait();
console.log(`${result.chunks.length} chunks, ${result.wordCount} words`);
